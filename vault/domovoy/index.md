# Домовой

Локальный голосовой ассистент для умного дома на русском языке. Вся обработка происходит на устройстве — голос не покидает домашнюю сеть.

---

## Ключевые принципы

- **Полностью локально** — работает без интернета и облачных сервисов
- **Приватность** — голосовые данные остаются дома
- **Русский язык** — нативная поддержка речи и синтеза
- **Открытый стек** — Home Assistant, открытые ML-модели

---

## Текущие возможности

### Голосовое управление

- Активация по wake word **«Домовой»** — кастомная модель, обученная на русском произношении
- Детекция речи (VAD) — запись заканчивается автоматически при паузе ~1.5 сек, не по таймеру
- Режим разговора — после ответа ассистент продолжает слушать без повторного wake word
- Прерывание ответа (barge-in) — можно говорить пока ассистент отвечает

### Распознавание речи (STT)

- Модель **GigaAM** — лучшее качество для русского языка, WER 8.4%
- Пунктуация из коробки, нет галлюцинаций на тишине

### Синтез речи (TTS)

- **Silero TTS v5**, голос Xenia
- Нормализация текста: числа, даты, температура (`°C`), скорость (`км/ч`), время (`HH:MM`), проценты

### Интеллект (LLM)

- Модель **Qwen2.5:3b** через Ollama — локально, без API-ключей
- История диалога с кольцевым буфером (до 10 сообщений, TTL сброс)
- Tool calling — LLM может вызывать функции:
  - Текущее время и дата
  - Погода (Open-Meteo API — единственный внешний запрос)

### Логирование и метрики

- JSONL-лог диалогов: session_id, текст пользователя/ассистента, латентность по каждому этапу
- Метрики пайплайна: STT latency, LLM latency, TTS latency, общее время

## Запланировано

> Полное описание каждой задачи → [[implementation-plan]]

### Ближайшее (P1)

- **Стоп-слова** — «стоп», «хватит», «замолчи» завершают диалог
- **Повтори** — «повтори», «ещё раз» воспроизводят последний ответ
- **Шифрование логов** — диалоги шифруются AES-256-GCM at rest
- **Сетевая изоляция** — ML-сервисы недоступны снаружи Docker-сети
- **Secrets management** — токены и ключи через Docker secrets

### Среднесрочно (P2)

- **Web Dashboard** — статус сервисов, история диалогов
- **Speaker Recognition** — идентификация пользователя по голосу, персонализация
- **mTLS** — шифрованный канал между hub и spoke устройствами

### Hub-Spoke (P3)

- **Multi-room** — несколько устройств (spoke) с одним вычислительным центром (hub)
- **OTA обновления** — spoke обновляются автоматически через hub
- **Rate limiting** — защита hub от злоупотреблений

### Идеи (Backlog)

- Home Assistant интеграция (управление умным домом)
- Two-stage LLM routing (лёгкая + тяжёлая модели)
- Wake Word v2 (дообучение на реальных голосах)
- Контекстная память (20 сек между командами)
- Kokoro TTS — кастомный голос
- NPU оптимизация на Orange Pi 5 (RK3588S)
- Детский режим

## Железо

**Текущий стенд:** Windows + Docker Desktop (разработка)

**Целевое устройство:** Orange Pi 5 (RK3588S, 8GB RAM)
- NPU 6 TOPS для ускорения ML-моделей
- Доступен в России
- Достаточно для всего стека на одной плате

**Hub-Spoke:** Orange Pi 5 как hub + мелкие SBC (Orange Pi Zero 3, Raspberry Pi 3B) как spoke в комнатах
