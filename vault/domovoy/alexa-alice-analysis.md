# Alexa и Алиса — Анализ функционала относительно Домового

> Источник: исследование возможностей Alexa+ (2025–2026) и Яндекс Алиса (2025).
> Цель: понять что делают флагманские ассистенты, и что из этого стоит реализовать в Домовом.

---

## Что у нас уже есть

| Компонент | Alexa / Алиса | Домовой |
|---|---|---|
| Wake word | Аппаратный чип, облако | Silero VAD (локально, любая речь) |
| STT | Облако (Amazon / Яндекс SpeechKit) | faster-whisper (локально) |
| TTS | Облако, фирменные голоса | Piper Irina (локально) |
| LLM | Amazon Nova / YandexGPT 5 | Ollama qwen2.5:3b (локально) |
| Смарт-хоум | Zigbee, Matter, тысячи устройств | Home Assistant (планируется) |
| Приватность | **Всё в облаке** (у Alexa — обязательно с марта 2025) | **100% локально** — главное преимущество |

---

## Функциональные области

### 1. Wake Word / Обнаружение речи

**Что делают они:**
- Alexa: выделенный чип на устройстве, точное распознавание "Alexa"
- Алиса: аппаратное обнаружение "Алиса", аудио идёт в облако на подтверждение

**Что у нас:**
- Silero VAD — срабатывает на любую речь, не требует конкретного слова
- Шаг 21 плана: обучить OpenWakeWord на "Домовой" (2–4 часа в Colab)

**Что стоит добавить:**
- [ ] Кастомное слово-триггер "Домовой" (Шаг 21) — уже в плане
- [ ] Порог чувствительности через UI, а не env (Шаг 23)

---

### 2. STT — Распознавание речи

**Что делают они:**
- Алиса: Яндекс SpeechKit — лучшее качество для русского среди всех коммерческих систем
- Alexa: облачный Amazon STT, умеренное качество для русского

**Что у нас:**
- faster-whisper small — хорошее качество для русского, но медленно (~8–12с на CPU)

**Что стоит добавить:**
- [ ] Шаг 18: Streaming STT — отдавать частичные результаты, снизить задержку
- [ ] Шаг 22: NPU-ускорение на Orange Pi 5 (NPU RK3588S → 2–3с вместо 10с)
- [ ] Тест whisper medium/large для повышения точности (если латентность позволяет)

---

### 3. TTS — Синтез речи

**Что делают они:**
- Alexa+: несколько голосов, 3 стиля личности (Brief / Chill / Sweet)
- Алиса: фирменный голос Алисы, эмоциональный синтез

**Что у нас:**
- Piper Irina — качественный голос, но стандартный датасет

**Что стоит добавить:**
- [ ] Шаг 20: Kokoro TTS на датасете RUSLAN — уникальный мужской голос
- [ ] Возможность выбора голоса через env уже есть (TTS_VOICE=irina/ruslan)

---

### 4. Диалог и контекст

**Что делают они:**
- Alexa+: сессионный контекст, не нужно повторять wake word между фразами (синяя подсветка = слушает), поддержка multi-turn диалога
- Алиса: полный контекст сессии через Alice AI LLM, синхронизация между устройствами и мобильным приложением, окно контекста 32к токенов

**Что у нас:**
- LLM (`qwen2.5:3b`) отвечает на каждый запрос независимо, без истории

**Что стоит добавить:**
- [ ] История диалога в LLM-клиенте: хранить последние N сообщений и передавать как `messages[]`
- [ ] `keep_alive` уже настроен (10 минут), модель остаётся в памяти между запросами
- [ ] Опциональный "follow-up mode": после ответа система снова слушает без wake word

---

### 5. Управление умным домом

**Что делают они:**
- Alexa: тысячи совместимых устройств, Routines (сложные автоматизации), Hunches (ИИ-предсказания привычек), Matter/Zigbee/Thread нативно
- Алиса: собственная экосистема устройств, Zigbee-хаб встроен в станцию, **офлайн сценарии через Zigbee без интернета**

**Что у нас:**
- Home Assistant как бэкенд — это правильное решение, покрывает всё что умеют Alexa и Алиса
- HA поддерживает те же протоколы (Zigbee2MQTT, Matter, Z-Wave и т.д.)

**Что стоит добавить:**
- [ ] Шаги 11–12–13: Regex intent parser + HA REST client — ближайший приоритет
- [ ] Шаг 19: LLM как intent-парсер (JSON output) — для нестандартных команд
- [ ] Ответ системы с состоянием устройств: "Свет в спальне уже включён"
- [ ] Area targeting: называть комнату → HA находит все устройства в этой области

---

### 6. Таймеры, напоминания, будильники

**Что делают они:**
- Alexa: несколько именованных таймеров одновременно, напоминания с геолокацией, Alexa+ сам предлагает время готовки по описанию блюда
- Алиса: базовые таймеры/будильники, Alice AI превращает поток мыслей в структурированные напоминания

**Что у нас:**
- Ничего — всё идёт в LLM → произвольный ответ

**Что стоит добавить:**
- [ ] HA-интеграция с `input_datetime` entities для таймеров
- [ ] Или простой встроенный таймер через `time.AfterFunc` + TTS-уведомление
- [ ] Хранение напоминаний в HA calendar entity

---

### 7. Воспроизведение музыки / медиа

**Что делают они:**
- Alexa: Spotify, Apple Music, Amazon Music, подкасты, аудиокниги, мультикомнатный звук
- Алиса: Яндекс Музыка (глубокая интеграция), подкасты, Кинопоиск, управление TV

**Что у нас:**
- Через HA media_player entity — теоретически уже поддерживается при HA-интеграции

**Что стоит добавить:**
- [ ] После шага 12 (HA client): голосовое управление `media_player` — включить музыку, пауза, следующий трек
- [ ] Пример: "Включи музыку" → HA `media_player.media_play`
- [ ] Пример: "Поставь рок" → HA `media_player.play_media` с поисковым запросом

---

### 8. Агентное поведение (Agent Actions)

**Что делают они:**
- Alexa+: Nova Act — визуальный агент, умеет открывать браузер, заполнять формы, делать заказы там где нет API
- Алиса: Function calling — разбивает запрос на API-вызовы, бронирует такси, заказывает еду

**Что у нас:**
- LLM отвечает текстом, никаких действий кроме HA

**Что стоит добавить:**
- [ ] Шаг 19: LLM возвращает структурированный JSON с action → выполняем действие в HA
- [ ] Потенциально: LLM function calling через Ollama (поддерживается в qwen2.5)
- [ ] Пример: "что-то холодно" → LLM: `{action: set_temperature, area: bedroom, value: 22}` → HA

---

### 9. Списки покупок / Заметки / Память

**Что делают они:**
- Алиса: "Моя память" — хранилище заметок и списков, доступно с любого устройства
- Alexa: списки синхронизируются с приложением, интеграция с Amazon корзиной

**Что у нас:**
- Ничего

**Что стоит добавить:**
- [ ] HA `todo` entity (Shopping List интеграция) — добавить в список через голос
- [ ] Или простой текстовый файл / SQLite-хранилище на устройстве

---

### 10. Мультикомнатный звук

**Что делают они:**
- Alexa: группы устройств, синхронное воспроизведение, пространственный звук
- Алиса: базовая мультикомнатная поддержка через несколько Яндекс Станций

**Что у нас:**
- Одно устройство, одна точка воспроизведения

**Что стоит добавить:**
- [ ] При масштабировании: несколько экземпляров Домового с общим HA-бэкендом
- [ ] Синхронизация через HA (media_player group)

---

### 11. Приватность

**Что делают они:**
- Alexa: **с марта 2025 весь звук обязательно в облаке**, локальная обработка убрана
- Алиса: потребительская версия — облако; корпоративная SpeechKit — on-premise

**Что у нас:**
- **100% локальная обработка** — это главное конкурентное преимущество Домового
- Нет зависимости от внешних серверов, работает без интернета

**Продвигать как фичу:**
- При Шаге 23 (Web UI): явно показывать "всё обрабатывается локально"

## Что Домовой делает лучше, чем Alexa и Алиса

1. **Полная приватность** — ни один байт аудио не покидает дом (в отличие от Alexa с марта 2025)
2. **Работа без интернета** — все ML-сервисы локальные
3. **Открытый код** — полный контроль над пайплайном, любая кастомизация
4. **Любой HA-совместимый девайс** — не ограничены экосистемой одного вендора
5. **Русскоязычный голос** — Piper Irina качественнее стандартных голосов Alexa для русского

---

## Что пока лучше у Alexa / Алисы

1. **Скорость отклика** — облако быстрее; у нас 10–15с на CPU (→ решается NPU на Orange Pi 5)
2. **Ширина экосистемы** — тысячи скиллов vs. всё сделано вручную
3. **Качество NLU для русского** — Яндекс SpeechKit лучше faster-whisper для акцентной/быстрой речи
4. **Агентное поведение** — Alexa Nova Act умеет браузер; у нас только HA API
5. **Мобильное приложение** — у них есть, у нас нет
