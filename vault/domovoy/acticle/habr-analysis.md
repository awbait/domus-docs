# Анализ статей с Хабра: что полезно для Домового

## Источники
1. **empenoso** — планирование замены Алисы на локального ассистента (HA + ESP32 + Whisper + Piper)
2. **Wiren Board / BARY** — рабочий локальный ассистент с кастомной моделью эмбеддингов на Orange Pi 5 Pro

---

## 1. Wake word — тренировка своей модели OpenWakeWord

**Самое ценное для нас прямо сейчас.**

Пользователь **splastunov** поделился рабочим пайплайном:
- Colab-ноутбук: https://colab.research.google.com/drive/1q1oe2zOyZp7UsB3jJiQ1IFn8z5YfjwEb
- GitHub: https://github.com/splastunov/openwakeword-ru-model-train
- Натренировал "Кузя" — работает 6-7 из 10

Процесс:
1. Генерация сэмплов через **Piper** (4 голоса, ~5000 штук) + записи реальных голосов
2. Клонирование голоса через **qwen3-tts** для реалистичных сэмплов (100-150 на голос)
3. Скрипт скачивает шумы, смешивает, обучает модель
4. На выходе — `.onnx` модель, совместимая с OpenWakeWord

**Вывод:** можно натренировать "Домовой" как wake word. Наша инфраструктура (OpenWakeWord сервер) уже поддерживает кастомные модели.

## 2. STT — альтернативы Whisper

| Движок | Скорость | Качество RU | Стриминг | Заметки |
|--------|----------|-------------|----------|---------|
| **faster-whisper** (наш) | Средняя | Хорошее | Нет | Текущий выбор |
| **VOSK** | Быстрый | Хорошее | Да | Несколько комментаторов рекомендуют, работает на слабом железе |
| **GigaAM** | Быстрый | ~90% сложных фраз | Да (через VAD) | Полностью локальный |
| **T-one** (t-tech) | Быстрый | Отличное | Да | HuggingFace, нужно 8GB RAM |

**VOSK** — самый упоминаемый как быстрый и качественный для русского. BARY использует VOSK и компенсирует его огрехи через обучение эмбеддингов на искажённых фразах.

**Вывод:** можно попробовать VOSK как альтернативу для Orange Pi (меньше ресурсов, стриминг). Но faster-whisper точнее — оставить как основной.

## 3. Intent Recognition — подход BARY

BARY не использует LLM для команд умного дома. Вместо этого:
- Кастомная **embedding model** (5M параметров, 128-dim вектор)
- Датасет ~16 000 команд (49 устройств, 18 помещений)
- **Поиск по сходству** вместо LLM-инференса
- Скорость: **~150ms** на Orange Pi 5 Pro CPU

Уровни датасета:
1. Базовые команды (включи/выключи)
2. Синонимы (вруби, зажги, подсветку сделай)
3. Склонения + типичные ошибки STT
4. Эмоциональные/разговорные формулировки
5. Контекстные короткие фразы (память 20 сек: "а потусклее")

Комментарий **artmel**: по факту это векторная база, не LLM. Предложен **гибридный подход**: эмбеддинги для 99% команд (быстро), LLM-fallback для сложных случаев.

**Вывод:** наш текущий подход (LLM qwen2.5:3b) — противоположный. LLM гибче, но медленнее. Когда дойдём до HA-интеграции, стоит рассмотреть гибрид: regex/эмбеддинги для частых команд, LLM для остального. Это уже заложено в архитектуре (`internal/intent/` в CLAUDE.md).

## 4. TTS

| Движок | Скорость | Качество | Ресурсы |
|--------|----------|----------|---------|
| **Piper** (наш) | Быстрый | Хорошее | Лёгкий |
| **Silero TTS** | Быстрый | Хорошее | Лёгкий |
| **XTTSv2** | Медленный | Отличное | 5GB VGPU |
| **Terra TTS + RUAccent** | Средний | Отличное | Средние |

BARY использует **короткие звуковые сигналы** вместо TTS для подтверждений ("свет включён" → просто бип). Голосовые ответы только для данных ("Температура 22 градуса") и уточнений.

**Вывод:** идея с бипами для подтверждений хороша — быстрее, не раздражает. Можно добавить позже.

## 5. Микрофоны

Рейтинг по отзывам из комментариев:
1. **ReSpeaker USB Mic Array V2.0** — микрофонный массив, шумоподавление, beam forming. Слышит wake word даже при воспроизведении музыки на 2-3м
2. **ReSpeaker Lite** — 2 микрофона, хорошо для wake word
3. **HA Voice Preview Edition** — 4 микрофона (на базе ReSpeaker), лучше ATOM Echo
4. **ESP32-S3-BOX** — средне, зависит от расстояния
5. **M5Stack ATOM Echo** — слабый микрофон, слабый динамик

**Вывод:** для Orange Pi — **ReSpeaker USB Mic Array** выглядит лучшим вариантом.

## 6. Архитектурные идеи

### Voice ID (идентификация по голосу)
BARY различает членов семьи и персонализирует: кому +22°, кому +25°. Потенциально интересно, но не приоритет.

### Контекстная память (20 сек)
После команды "включи свет в спальне" можно сказать "а потусклее" в течение 20 секунд. Реализуется через хранение последнего контекста в оркестраторе.

### Гибридный STT+LLM
Комментарий **Asm0deus**: N150 + HA + n8n + Whisper + OpenRouter (LLM). LLM в промпте получает задачу определить объект + действие. Понимает "Включи над головой на кухне", "выруби все", "чёт над столом темно".

### Omni-модели
Несколько комментаторов упоминают мультимодальные модели (Qwen omni, Voxtral от Mistral) которые принимают голос напрямую, без STT. Пока сырое, но направление будущего.
