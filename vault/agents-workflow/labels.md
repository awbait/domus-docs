# Labels — спецификация

Минимальная схема. Все labels версионируются в `.gitea/labels.yml` (раскатывается на все репо через `tea labels create`).

## Тип задачи (`type:*`) — обязательно один

| Label              | Цвет      | Описание                        |
| ------------------ | --------- | ------------------------------- |
| `type:feature`     | `#0e8a16` | Новая функциональность          |
| `type:bug`         | `#d73a4a` | Баг                             |
| `type:enhancement` | `#a2eeef` | Улучшение существующего         |
| `type:chore`       | `#fef2c0` | Рефакторинг, зависимости, инфра |
| `type:docs`        | `#0075ca` | Документация                    |
| `type:research`    | `#d4c5f9` | Исследование, без кода          |

## Статус (`status:*`) — ровно один на issue

| Label                   | Цвет      | Описание                                  |
| ----------------------- | --------- | ----------------------------------------- |
| `status:triage`         | `#ededed` | Новое, нужна разметка triager'ом          |
| `status:needs-analysis` | `#c5def5` | Требует анализа агентом перед реализацией |
| `status:needs-approval` | `#fbca04` | Анализ готов, ждёт approval от человека   |
| `status:ready`          | `#0e8a16` | План одобрен, можно брать в работу        |
| `status:in-progress`    | `#1d76db` | Взято агентом или человеком               |
| `status:review`         | `#d93f0b` | PR создан, ждёт review                    |
| `status:blocked`        | `#b60205` | Заблокировано (причина в комментарии)     |

Закрытое issue = done, отдельный label не нужен.

## Приоритет (`priority:*`) — опционально

| Label         | Цвет      | Описание                |
| ------------- | --------- | ----------------------- |
| `priority:p0` | `#b60205` | Блокер, бросаем всё     |
| `priority:p1` | `#d93f0b` | Этот спринт             |
| `priority:p2` | `#fbca04` | Когда дойдут руки       |

## Спецфлаги

| Label        | Описание                                                    |
| ------------ | ----------------------------------------------------------- |
| `human-only` | **Запрещает** делегирование агенту (security, архитектура). По умолчанию все задачи могут быть делегированы. |
| `refactor`   | Триггер для refactor-агента (вместе с `type:chore`)         |

## Что НЕ используем (и почему)

| Убрано        | Причина                                                                   |
| ------------- | ------------------------------------------------------------------------- |
| `agent:*`     | Дефолт = можно делегировать. Запрет — единственный флаг `human-only`     |
| `scope:*`     | Gitea Issues per-repo, репо сам по себе = scope                          |
| `area:*`      | Лишняя детализация. Агент сам разберётся какие файлы трогать по тексту   |
| `parallel:*`  | Worktree isolation решает изоляцию автоматически                         |
| `needs-tests` | Тесты — часть acceptance criteria, не отдельный label                    |
| `release`     | Релизы делает человек                                                    |

## Правила

1. **Ровно один `type:*`** на issue.
2. **Ровно один `status:*`** на issue в любой момент.
3. **`priority:*`** опционален, по умолчанию — без приоритета (~p2).
4. **Закрытое issue** не нужно лейблить дополнительно — закрытие = done.
5. **При закрытии без merge** — комментарий с причиной.
6. **`human-only`** ставится только когда задача требует решений уровня архитектуры/безопасности/стратегии, которые нельзя делегировать.

## Создание labels в Gitea

```bash
# Через tea CLI (для каждого репо отдельно — Gitea labels per-repo):
tea labels create --name "type:feature" --color "#0e8a16"
tea labels create --name "type:bug" --color "#d73a4a"
# ... и так далее
```
